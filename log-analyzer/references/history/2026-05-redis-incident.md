# 장애 이력: 2026-05 Redis 타임아웃 (JIRA-1567)

- **일시**: 2026-05-22 19:10 ~ 19:38 (28분)
- **증상**: cache-service redis timeout 반복 → api-gateway 504 → 로그인/세션 조회 실패 다수
- **원인**: 트래픽 피크에 redis 커넥션 풀(50)이 부족해 wait_queue가 쌓임. 일부 슬로우 명령(대용량 SCAN)이 풀 점유 시간을 늘려 악화.
- **해결**: redis pool size 50 → 100 증설 + 문제 SCAN을 배치 분할로 변경. 이후 재발 없음.
- **교훈**:
  - DB 풀 고갈과 로그가 닮았지만 **자원이 redis라 대응이 완전히 다르다** — 자원 식별자부터 확인.
  - `wait_queue > 0`은 pool active 수치보다 빠른 선행 신호 — 90% 도달 전에 큐부터 본다.
  - upstream 504는 결과일 뿐 — 알림은 원인(redis 풀/슬로우 명령)을 짚어줘야 대응이 빠르다.

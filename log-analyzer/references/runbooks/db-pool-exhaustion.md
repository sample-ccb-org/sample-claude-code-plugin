# Runbook: DB 커넥션 풀 고갈 대응

## 즉시 (5분 내)

1. 현재 풀 상태 확인: `pool_in_use` 최신 수치, timeout 발생률
2. **pool size가 50 미만이면 즉시 증설** (2026-03 장애의 검증된 해결책):
   - 설정 `db.pool.max_size=50` → 설정 리로드 (재시작 불필요)
3. payment-service 에러율 확인 — downstream 전파 여부

## 후속 (당일)

- 새벽 배치와 트래픽 피크가 겹치는지 스케줄 확인
- 풀 사용률 80% 경고 알림이 동작했는지 점검

## 에스컬레이션

- 증설 후에도 timeout이 계속되면 DB 자체 병목(슬로우 쿼리) 의심 — DBA 호출
